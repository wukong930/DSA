# 📈 A股/港股/美股自选股智能分析系统 — 产品技术报告

---

## 一、产品概述

**Daily Stock Analysis（DSA）** 是一套面向个人投资者和量化爱好者的 AI 驱动股票智能分析系统。系统覆盖 A 股、港股、美股三大市场，通过多维度数据采集、技术指标计算、新闻情报检索与大语言模型（LLM）深度分析，自动生成包含买卖决策、操作清单和风险提示的投资分析报告，并支持多渠道推送通知。

系统提供 Web 工作站、桌面客户端（Electron）、Bot 交互（Telegram/飞书/钉钉/Discord）、GitHub Actions 定时执行等多种使用方式，实现零成本或低成本的自动化投资分析。

## 二、核心功能矩阵

### 2.1 AI 决策看板

| 能力 | 说明 |
|------|------|
| 一句话结论 | 每只股票生成信号类型（看多/看空/观望）+ 仓位建议 |
| 精准狙击点 | 基于技术分析给出具体买入/卖出价位 |
| 操作清单 | 可执行的交易策略，含止损止盈位 |
| 风险提示 | 结合新闻情报和技术面的风险预警 |

### 2.2 多维度分析引擎

- **技术面分析**：43 项技术指标（MA/EMA/MACD/RSI/KDJ/BOLL/ATR/OBV 等），支持多头排列、金叉死叉、趋势判断
- **筹码分布分析**：成本集中度、获利盘比例、套牢盘分布
- **基本面聚合**：估值（PE/PB/PS）、成长性、盈利能力、机构持仓、资金流向、龙虎榜、板块概念
- **情报面**：多源新闻检索 + 舆情分析（Reddit/X/Polymarket 情绪数据）
- **实时行情**：盘中实时价格计算技术指标，非交易时段使用历史收盘价

### 2.3 市场策略系统

- **A 股三段式复盘**：大盘概况 → 板块轮动 → 资金流向，生成每日市场综述
- **美股 Regime Strategy**：基于市场状态（牛市/熊市/震荡）的策略切换
- **全球指数覆盖**：SPX、DJI、IXIC 等主要指数

### 2.4 Agent 智能问答

基于 Agent 框架的交互式股票研究助手：

- **11 套内置策略**：趋势跟踪、均值回归、动量策略、价值投资等
- **多专家协作**：决策 Agent、技术 Agent、情报 Agent、风险 Agent、组合 Agent
- **工具调用**：Agent 可自主调用数据获取、技术分析、新闻搜索、回测等工具
- **流式对话**：支持 SSE 实时流式输出分析过程

### 2.5 AI 回测验证

- 基于历史数据验证分析准确性
- 支持自定义回测区间和策略参数
- 生成回测绩效报告（胜率、收益率、最大回撤等）

### 2.6 持仓管理

- 多账户管理（支持不同券商/市场）
- 交易记录导入（CSV/Excel，支持多券商格式自动识别）
- 持仓快照与盈亏计算
- 现金流水与公司行动（分红/拆股）记录
- 汇率自动刷新（跨市场持仓）
- 风险分析（集中度预警、回撤预警、止损预警）

### 2.7 自选股与监控

- 自选股管理（添加/删除/标签/备注）
- 条件筛选（基于技术指标的多条件组合筛选）
- 实时监控任务（自定义监控条件 + 触发间隔）
- 告警历史记录

### 2.8 定时任务调度

- 每日定时分析
- 固定间隔执行
- 工作日定时（自动跳过非交易日）
- GitHub Actions 云端调度（零服务器成本）

### 5.2 多渠道 Fallback

系统支持配置多个 LLM 渠道，实现：
- **多 Key 轮询**：同一提供商多个 API Key 负载均衡
- **跨提供商 Fallback**：主渠道失败自动切换备用渠道
- **运行时热配置**：通过 Web 设置页面动态调整，无需重启

## 八、安全设计

### 8.1 认证体系

- **可选认证**：默认关闭，可通过 Web 设置页面启用
- **单用户模式**：文件级凭证，适合个人使用
- **多用户模式**：数据库级用户管理，支持角色（admin/analyst）
- **Cookie Session**：HttpOnly Cookie，防 XSS
- **登录限流**：429 状态码防暴力破解

### 8.2 API 安全

- CORS 白名单控制
- Auth 中间件统一拦截 `/api/v1/*`
- 豁免路径：健康检查、登录、API 文档
- 请求体大小限制（20MB）

### 8.3 数据安全

- 敏感配置通过环境变量注入，不硬编码
- `.env` 文件不入版本控制
- Docker 容器网络隔离
- HTTPS 加密传输（Let's Encrypt）

## 十四、技术风险与改进建议

### 14.1 当前风险

| 风险 | 等级 | 说明 |
|------|------|------|
| 数据源稳定性 | 中 | 免费数据源存在反爬限制，东财接口频繁断连 |
| LLM 成本 | 低-中 | 大量分析时 API 调用费用可能上升 |
| 单点故障 | 中 | 单容器部署，无高可用方案 |
| 数据一致性 | 低 | 多源数据可能存在微小差异 |

### 14.2 改进建议

1. **缓存层**：引入 Redis 缓存热点行情数据和 LLM 响应，降低 API 调用频率
2. **消息队列**：引入 Celery/RabbitMQ 解耦分析任务，提升并发能力
3. **监控告警**：集成 Prometheus + Grafana 监控系统健康状态
4. **数据持久化**：行情数据本地持久化，减少重复拉取
5. **移动端**：基于现有 Web 技术栈开发 PWA 或 React Native 移动应用
